# Static QR Code System

Generate permanent QR codes that customers can scan to initiate payments with amounts they choose themselves. Perfect for table service, tip collection, donations, and flexible payment scenarios.

## How Static QR Codes Work

### Customer-Initiated Payments

<img src="/images/static-qr-overview.png" alt="Static QR Code Overview" style={{borderRadius: '8px', border: '1px solid #e1e5e9'}} />

**The Process:**
1. **Merchant Generates**: Create a permanent QR code for your business
2. **Customer Scans**: Customer uses any QR scanner or banking app
3. **Amount Entry**: Customer enters their desired payment amount
4. **Payment Flow**: Standard secure payment processing through Token.io
5. **Confirmation**: Both parties receive payment confirmation

**Key Benefits:**
- **Flexible Amounts**: Customers choose how much to pay
- **No Staff Required**: Self-service payment option
- **Multiple Uses**: One QR code for all transactions
- **Quick Setup**: Generate once, use indefinitely

## QR Code Generation

### Creating Your Static QR Code

<img src="/images/qr-generation.png" alt="QR Code Generation" style={{borderRadius: '8px', border: '1px solid #e1e5e9'}} />

**Generation Process:**
```javascript
const staticQRConfig = {
  merchantId: "merchant_123",
  businessName: "Your Coffee Shop",
  currency: "EUR",
  logo: "https://yourbusiness.com/logo.png",
  description: "Pay Your Coffee Shop",
  features: {
    variableAmount: true,
    tipOption: true,
    customerInfo: false
  }
};
```

**QR Code Features:**
- **Business Branding**: Include your logo and business name
- **Currency Selection**: Specify your preferred currency
- **Custom Description**: Add payment purpose description
- **Security Token**: Unique encrypted identifier for your business

## Technical Specifications

### QR Code Standards

**Technical Details:**
- **Format**: QR Code 2005 standard
- **Error Correction**: Level H (30% recovery)
- **Encoding**: UTF-8 for international support
- **Size**: Minimum 2x2 cm for reliable scanning

**Code Structure:**
```
https://pay.acountpay.com/s/[merchant_token]
```

**Token Contents:**
```javascript
const qrToken = {
  merchantId: "merchant_123",
  tokenType: "static",
  issuedAt: "2024-01-15T10:30:00Z",
  expiresAt: null, // Never expires
  features: ["variable_amount", "tips"],
  version: "1.0"
};
```

### Security Features

**Security Measures:**
- **Encrypted Tokens**: Merchant information encrypted
- **HTTPS Required**: All transactions over secure connections
- **Rate Limiting**: Prevent abuse and fraud attempts
- **Fraud Detection**: Automatic suspicious activity monitoring

**Security Configuration:**
```javascript
const securityConfig = {
  encryption: {
    algorithm: "AES-256",
    tokenRotation: false, // Static tokens don't rotate
    signatureVerification: true
  },
  rateLimiting: {

    maxAmountPerTransaction: 1000.00,
    suspiciousActivityThreshold: 10
  },

    fraudDetection: true,
    geolocationTracking: false,

  }
```

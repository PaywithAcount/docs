---
title: "Setup the SDK"
---

After installing the AcountPay SDK, follow these steps to set it up in your project.

## Prerequisites

- Installed AcountPay SDK (see [Install the SDK](/sdk/install_sdk))
- Your AcountPay Client ID from the merchant dashboard
- API Base URL (e.g., 'https://acount-apis-7886a9e16dc3.herokuapp.com' for production)

## Initialization

Initialize the SDK with your client ID and environment settings.

### Basic Setup

```typescript
import AcountPay from '@acountpay/sdk';

// Initialize the SDK
const sdk = new AcountPay({
  clientId: 'your-client-id-here', // Required: From merchant dashboard
  env: 'sandbox', // Optional: 'sandbox' or 'production' (default: 'production')
  apiBaseUrl: 'https://acount-apis-7886a9e16dc3.herokuapp.com' // Optional: Override default API URL
});
```

### UMD Build Setup (Script Tag)

If using the UMD build, initialize without importing:

```html
<script src="https://unpkg.com/@acountpay/sdk@1.0.0/dist/acount.umd.js"></script>
<script>
  // Runtime configuration (optional)
  window.__ACOUNTPAY_API_URL__ = 'https://acount-apis-7886a9e16dc3.herokuapp.com';
  window.__ACOUNTPAY_ENV__ = 'sandbox';

  // Initialize the SDK
  const sdk = new Acount({
    clientId: 'your-client-id-here'
  });
</script>
```

## Configuration Options

| Option       | Type     | Required | Description |
|--------------|----------|----------|-------------|
| `clientId`   | `string` | ✅ Yes   | Your AcountPay client ID |
| `env`        | `string` | ❌ No    | Environment: 'sandbox' or 'production' (default: 'production') |
| `apiBaseUrl` | `string` | ❌ No    | Custom API base URL (overrides default) |

## Runtime Configuration (UMD Only)

For UMD builds, you can set global variables before loading the script:

- `window.__ACOUNTPAY_API_URL__`: API base URL
- `window.__ACOUNTPAY_ENV__`: Environment ('sandbox' or 'production')

## Testing Setup

To test in sandbox mode:

```typescript
const sdk = new AcountPay({
  clientId: 'your-sandbox-client-id',
  env: 'sandbox'
});
```

## Next Steps

Once set up, you can initiate payments. See [Initialize Payment](/sdk/initialize_payment) for examples.

## Troubleshooting

- **"AcountPay is not defined"**: Ensure the UMD script is loaded before initialization.
- **Invalid client ID**: Verify your client ID from the merchant dashboard.
- **API errors**: Check your API base URL and environment settings.

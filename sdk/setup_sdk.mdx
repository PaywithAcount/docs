Before making any method calls, you must initialize the SDK with your account credentials.

The `Account` class is responsible for initializing the SDK with authentication credentials. It requires a **Client ID** to be provided during instantiation, ensuring that all subsequent API requests are authenticated.

***

### **How to Initialize the SDK**

To use the SDK, you must first **initialize** the `Account` class with a valid `clientId`.

### **Example Usage**

```typescript
import Account from "acount-sdk";

const sdk = new Account({
  clientId: "your-client-id-here",
});
```

### **Error Handling**

If an invalid or missing `clientId` is provided, the following error will be thrown:

```typescript
try {
  const account = new Account({ clientId: "" }); // Missing Client ID
} catch (error) {
  console.error(error.message); // Output: "Client ID is required"
}
```

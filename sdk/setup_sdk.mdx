Before making any method calls, you must initialize the SDK with your account credentials.

The AcountPayClient class is responsible for initializing the SDK with authentication credentials. It requires a Client ID to be provided during instantiation, ensuring that all subsequent API requests are authenticated.

## Constructor Parameters

| Parameter | Type | Required | Description |
|-----------|------|----------|-------------|
| `clientId` | `string` | ‚úÖ Yes | The unique Client ID found in your merchant dashboard |
| `environment` | `string` | ‚úÖ Yes | Either `'sandbox'` (testing) or `'production'` (live) |
| `redirectUri` | `string` | ‚úÖ Yes | The URI where users will be redirected after authentication |
| `locale` | `string` | ‚ùå No | Language preference (`'da'` or `'en'`, defaults to `'da'`) |
| `theme` | `object` | ‚ùå No | UI customization options |

## Environment Variables

The following environment variables are recommended for your `.env` file:

```bash
# .env file
REACT_APP_ACOUNTPAY_CLIENT_ID=your-client-id-here
REACT_APP_ACOUNTPAY_ENV=sandbox
REACT_APP_ACOUNTPAY_REDIRECT_URI=https://your-domain.com/callback
```

## How to Initialize the SDK
To use the SDK, you must first initialize the AcountPayClient with a valid clientId.

Example Usage
```react
// Import the SDK
import { AcountPayClient } from '@acountpay/sdk';
```

```react
// Initialize with your credentials
const acountPay = new AcountPayClient({
  clientId: process.env.REACT_APP_ACOUNTPAY_CLIENT_ID,
  environment: process.env.REACT_APP_ACOUNTPAY_ENV || 'sandbox',
  redirectUri: process.env.REACT_APP_ACOUNTPAY_REDIRECT_URI,
  // Optional configuration
  locale: 'da', // Default language ('da' for Danish, 'en' for English)
  theme: {
    primaryColor: '#166534', // green-800 in Tailwind
    // Additional theming options
  }
});
```

## Error Handling
If an invalid or missing clientId is provided, the following error will be thrown:
```react
try {
  const acountPay = new AcountPayClient({ clientId: "" }); // Missing Client ID
} catch (error) {
  console.error(error.message); // Output: "Client ID is required"
}
```
## React Component Setup
If you're using React, you can use our provider component:
```react

import { AcountPayProvider } from '@acountpay/sdk/react';

function App() {
  return (
    <AcountPayProvider
      clientId={process.env.REACT_APP_ACOUNTPAY_CLIENT_ID}
      environment={process.env.REACT_APP_ACOUNTPAY_ENV}
      redirectUri={process.env.REACT_APP_ACOUNTPAY_REDIRECT_URI}
    >
      {/* Your application */}
      <YourApp />
    </AcountPayProvider>
  );
}
```
## Testing Your Setup
To verify your setup is working:
```react
// Check connection status
acountPay.checkConnection()
  .then(status => {
    console.log('AcountPay connection status:', status);
  })
  .catch(error => {
    console.error('AcountPay connection error:', error);
  });
```
